Основная идея проекта — создать **компонентную UI-систему для PHP с фокусом на иммутабельности и предсказуемости**.

**Ключевые концепции:**

1. **Иммутабельные компоненты (`Brick`)** — компоненты, которые после создания не изменяются. Это гарантирует предсказуемый рендеринг и безопасность в многопоточной среде.

2. **Разделение логики и представления** — каждый компонент состоит из:
    - PHP класса с данными
    - HTML шаблона (`template.php`)
    - CSS стилей
    - JavaScript кода

3. **Централизованное управление** — через `BrickManager`:
    - Кэширование компонентов и их ресурсов
    - Рендеринг CSS и JS всех компонентов
    - Статистика и отладка

4. **Модульность через трейты** — расширение функциональности:
    - `WithCache` — кэширование рендера
    - `WithHelpers` — вспомогательные методы для шаблонов
    - `WithStrictHelpers` — строгие проверки типов
    - `WithInheritance` — наследование шаблонов и ресурсов

5. **Гибкое управление ассетами** — разные стратегии рендеринга CSS/JS:
    - Встроенные в страницу (`InlineAssetRenderer`)
    - В отдельные файлы (`FileAssetRenderer`)
    - С минификацией и группировкой

6. **Мутабельная альтернатива** (`Clay`) — для особых случаев, когда требуется изменяемое состояние.

**Проблемы, которые решает система:**
- Предсказуемый HTML-вывод
- Безопасность от XSS (экранирование через `e()`)
- Эффективное кэширование
- Повторное использование компонентов
- Статический анализ и типобезопасность
- Легкая отладка и тестирование

**Идеология:** Создание UI-компонентов, которые ведут себя как математические функции — одинаковые входные данные всегда дают одинаковый HTML-вывод, что упрощает разработку, тестирование и отладку сложных интерфейсов.
